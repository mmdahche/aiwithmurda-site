# The Future Proof Method - Day 0-7 Stream Run Sheet

Use this for the first week. The job is not to fill time. The job is to create repeatable proof moments: setup, build, receipt, clip, offer, follow-up.

Rule: every live block should answer one of four questions:

- What are we trying to prove?
- What changed on screen?
- What did the scoreboard say?
- What should a viewer do next?

## Daily Show Loop

Run this every day, even if the actual stream length changes.

1. **Open loop** - state the day, the target, and the proof metric.
2. **Scoreboard check** - show revenue, followers, email, clips, hours, builds.
3. **Build block** - ship one visible slice with AI help.
4. **Friction block** - name what broke, stalled, or got overbuilt.
5. **Receipt block** - write before, after, proof, lesson, tomorrow.
6. **Clip block** - choose the strongest hook and title it.
7. **Offer block** - point to the kit only when the proof supports it.
8. **Shutdown block** - update the tracker and write tomorrow's promise.

## Day 0 - Setup Stream

Goal: make the system feel real before the sprint starts.

Proof target:
- `/60`, `/live`, `/tools`, `/kit`, `/members`, `/overlay`, `/obs`, and `/day/1` load.
- `npm run smoke:launch` passes.
- OBS browser source is framed.

Live beats:
- Show the public scoreboard.
- Show the command shelf.
- Explain the privacy rule: build mode is public, sensitive data goes private.
- Walk through the kit assets without opening private customer/payment data.
- End with the Day 1 promise.

Clip hooks:
- "I built the scoreboard before the sprint started."
- "The stream has commands before the stream has hype."
- "This is the system I am betting 60 days on."

CTA:
- `!start` for the build log.
- `!kit` only after showing the kit path.

## Day 1 - Baseline And First Receipt

Goal: replace preview/demo data with the clean Day 1 baseline.

Proof target:
- Baseline pushed with `npm run baseline:launch:push`.
- Day 1 receipt page is live.
- First build or setup proof exists.

Live beats:
- Show the baseline at zero or true starting numbers.
- Name the first offer lane.
- Complete Module 1 task one: command folders.
- Capture the first receipt.

Clip hooks:
- "Day 1 starts at zero, publicly."
- "I am not hiding the scoreboard."
- "This is what a clean AI sprint baseline looks like."

CTA:
- `!scoreboard` during baseline.
- `!today` after the Day 1 receipt is live.

## Day 2 - Problem Selection

Goal: choose a painful workflow small enough to prove quickly.

Proof target:
- 10 workflow candidates.
- Top 3 scored.
- One before-state receipt.

Live beats:
- Score workflows by proof speed, buyer pain, viewer clarity, and money path.
- Reject ideas that are too big.
- Pick one smallest useful build.
- Write the before-state receipt.

Clip hooks:
- "The idea was too big, so I cut it down."
- "This is how I choose what to vibe-code live."
- "If the pain is not visible, the build will not matter."

CTA:
- `!kit` after showing the Problem To Proof framework.

## Day 3 - First Narrow Build

Goal: ship one visible improvement.

Proof target:
- Scoped build brief.
- Working slice.
- Commit or handoff.

Live beats:
- Ask AI to inspect before changing code.
- Define the stop condition.
- Ship the smallest slice.
- Run the user path on screen.

Clip hooks:
- "Codex inspected first, then we built."
- "The first version worked because the scope was tiny."
- "This is the part people skip before blaming AI."

CTA:
- `!today` after receipt.
- `!kit` if the build demonstrates the module sequence.

## Day 4 - Proof Packaging

Goal: turn the build into public receipts and clips.

Proof target:
- Before/after receipt.
- One clip hook.
- One public post or scheduled asset.

Live beats:
- Show the before and after side by side.
- Name the failure.
- Write the clip hook.
- Decide where the proof points: scoreboard, kit, email list, or next stream.

Clip hooks:
- "The build was not done until the receipt existed."
- "Here is the before and after."
- "The failure made the clip better."

CTA:
- `!start` for people who want the daily receipts.

## Day 5 - Offer Tightening

Goal: make the $47 offer clearer from the week's proof.

Proof target:
- One improved CTA or promise.
- One objection answer.
- One follow-up message.

Live beats:
- Pick the strongest proof receipt.
- Ask what promise it makes more believable.
- Rewrite one CTA.
- Add one objection answer.
- Send or draft one warm follow-up.

Clip hooks:
- "Proof should change the offer, not just the content."
- "This receipt made the CTA sharper."
- "The $47 kit got better because the stream exposed the weak part."

CTA:
- `!kit` after the offer rewrite.

## Day 6 - Systems Cleanup

Goal: fix the rough edge that slowed the first five days.

Proof target:
- One workflow cleanup.
- One automation, template, route, or checklist improvement.
- One saved lesson for members.

Live beats:
- Review the biggest repeated friction.
- Pick one cleanup.
- Ship it.
- Add the lesson to the member system if it is reusable.

Clip hooks:
- "The boring fix is what makes tomorrow faster."
- "I cleaned the system instead of adding another feature."
- "This is how the kit improves during the sprint."

CTA:
- `!members` for buyers.
- `!kit` for non-buyers.

## Day 7 - Weekly Review

Goal: produce the first weekly recap and next bet.

Proof target:
- Best proof receipt.
- Biggest metric jump.
- Biggest miss.
- Offer change.
- Next week's main bet.

Live beats:
- Walk the scoreboard.
- Pick the best proof.
- Name the miss.
- Decide the offer change.
- Set the next weekly bet.

Clip hooks:
- "Week 1 numbers are in."
- "This was the strongest proof from the first week."
- "The scoreboard changed the plan."

CTA:
- `!scoreboard` during recap.
- `!kit` after the offer change.

## Weekly Clip Buckets

- Baseline: starting numbers, stakes, public dashboard.
- Build: AI handoff, scoped fix, before/after.
- Failure: broken assumption, overbuild, messy debugging.
- Lesson: one sentence that changes tomorrow.
- Offer: proof-backed promise, objection, CTA.
- Human: why the sprint matters, family away, disciplined grind.

## Shutdown Checklist

- [ ] Tracker updated.
- [ ] Day receipt written.
- [ ] Strongest clip named.
- [ ] Offer note updated.
- [ ] Tomorrow promise written.
- [ ] Sensitive screens closed before stream ends.
