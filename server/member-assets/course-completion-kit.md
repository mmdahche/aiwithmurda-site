# The Future Proof Method - Completion Kit

Finish with a verified build, not a watched playlist.

Completion means both agents are usable, the project is AI-ready, the operator loop has shipped a real change, starter skills are installed, and one useful build can be reproduced.

## Capstone - First Build Handoff

Package the five module outputs into one handoff another person or AI agent can use: setup, project rules, build brief, working result, verification, installed skills, and next move.

Output: One portfolio-ready build receipt and a practical handoff for the next session.

Capstone prompt:

> Turn my completed Future Proof Method outputs into a concise first-build handoff. Include tool setup, project purpose, instructions, shipped user path, verification evidence, installed skills, known limits, and the next smallest build.

## Completion criteria

- [ ] Both builders are ready
  - Evidence: Claude Code, Codex, and Git are verified inside one dedicated project.
- [ ] The project carries its own context
  - Evidence: Brief, instructions, commands, secret boundaries, and baseline checkpoint exist.
- [ ] The operator loop shipped a change
  - Evidence: Inspect, plan, build, verify, and checkpoint receipts exist for one real user path.
- [ ] Starter skills are installed
  - Evidence: Three reusable skills were tested and one was customized for the project.
- [ ] The first useful build is reproducible
  - Evidence: The core path works from a clean start and the README and handoff explain what comes next.
