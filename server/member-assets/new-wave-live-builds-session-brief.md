# New Wave Live Builds - Session Build Brief

## Purpose

New Wave Live Builds is a paid room where one real workflow gets turned into a working AI-assisted asset in public. The goal is not to teach theory. The goal is to show the operator loop under pressure:

1. Pick the workflow.
2. Scope the smallest useful slice.
3. Build with AI.
4. Test the user path.
5. Capture proof.
6. Turn the proof into an offer or content move.

## Room 001 Selection Score

Use this score before picking the first live build.

| Signal | Question | Score |
| --- | --- | --- |
| Buyer pain | Does this remove a real manual task or confusion? | 1-5 |
| Proof speed | Can we show before and after inside one session? | 1-5 |
| Stream clarity | Can viewers understand the stakes in under 60 seconds? | 1-5 |
| Money path | Does this connect to followers, emails, checkout, or buyers? | 1-5 |
| Reusability | Can buyers reuse the workflow after the room? | 1-5 |

Pick the highest-scoring workflow unless one option has a clear deadline.

## Stop Conditions

Stop building when one of these is true:

- A real user path works end to end.
- The before/after proof is clear enough for a receipt.
- The session hits the timebox and the remaining work becomes a documented backlog.

Do not chase polish until the proof exists.

## Session Receipt

End the room with:

- What was messy before.
- What changed after.
- The exact build slice shipped.
- The prompt or command that made the biggest difference.
- One mistake worth avoiding.
- One clip hook.
- One implementation step buyers can do this week.
