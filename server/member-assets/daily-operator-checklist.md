# Daily Operator Checklist

Use this every build day. Keep it boring. Boring systems create loud outcomes.

## Morning Setup

- [ ] Open the command center.
- [ ] Check yesterday's numbers.
- [ ] Pick one main goal for today.
- [ ] Pick one public proof target.
- [ ] Pick one money-path action.
- [ ] Clear the first blocker before checking social.

## Build Block

- [ ] Write the outcome in one sentence.
- [ ] Ask AI to inspect the existing project before changing anything.
- [ ] Make the smallest useful change.
- [ ] Test the actual user path.
- [ ] Capture before/after proof.
- [ ] Commit when the slice works.

## Stream Block

- [ ] Say what you are building.
- [ ] Show the scoreboard.
- [ ] Show the current blocker.
- [ ] Let the audience watch a real attempt.
- [ ] Summarize the lesson when the attempt succeeds or fails.
- [ ] Clip the highest-friction moment.

## Content Block

- [ ] Pull one clip from the build.
- [ ] Write one hook from the viewer's pain.
- [ ] Show proof inside the first 3 seconds.
- [ ] End with one next step.
- [ ] Post to the primary platform.
- [ ] Save the post URL in the proof folder.

## Money Path

- [ ] Improve one CTA.
- [ ] Message one warm lead.
- [ ] Add one buyer objection to the product page.
- [ ] Make one offer clearer.
- [ ] Follow up with one person.

## Shutdown

- [ ] Log metrics.
- [ ] Write best moment.
- [ ] Write biggest failure.
- [ ] Write lesson learned.
- [ ] Write tomorrow's promise.
- [ ] Save proof assets.
- [ ] Decide the first action for tomorrow.

## Daily Score

Give the day a score from 1 to 5:

- 1 = drifted
- 2 = worked but no proof
- 3 = shipped proof
- 4 = shipped proof and moved money path
- 5 = shipped, sold, learned, and set up tomorrow
