# Prompt Workflow Pack

These prompts are meant to be reused. Replace the bracketed parts with your context.

## 1. Business Problem Finder

```text
Act like a practical business operator. I run [business/project]. Find 20 repetitive, expensive, annoying, or slow workflows that could be improved with AI or simple software. For each one, give me:
- the pain
- who feels it
- what it costs
- the simplest version of a fix
- whether it can become content, a tool, or an offer
Rank them by speed to proof and money potential.
```

## 2. Workflow Mapper

```text
Map this workflow: [describe the workflow].

Break it into:
1. trigger
2. inputs
3. steps
4. decisions
5. outputs
6. failure points
7. manual work that AI can reduce
8. data that must be captured
9. simplest v1 build
10. proof metric

Keep it practical. Do not invent a big platform if a small tool would prove the point.
```

## 3. Offer Builder

```text
Turn this workflow fix into a paid offer:

Workflow: [workflow]
Audience: [audience]
Pain: [pain]
Outcome: [outcome]
Proof I have: [proof]

Give me:
- one $47 product
- one $97-$297 live offer
- one $1,000+ implementation offer
- the promise
- what is included
- what is not included
- who should not buy
- checkout page copy
```

## 4. Live Build Segment Planner

```text
I am going live to build [thing]. Turn this into a stream segment plan.

Create:
- opening 30 seconds
- scoreboard check
- build objective
- visible proof moment
- risk or challenge
- viewer interaction prompt
- clip-worthy moment to aim for
- closing recap

Keep it entertainment-first, but still useful.
```

## 5. Proof Extractor

```text
Here is what happened today: [paste notes].

Extract:
- best proof asset
- best hook
- best lesson
- strongest before/after
- one tweet/thread
- one TikTok/Reels script
- one email recap
- one scoreboard update
```

## 6. QA Before Shipping

```text
Review this change before I ship: [describe change or paste code/context].

Look for:
- broken user paths
- confusing copy
- missing error states
- privacy risks
- payment/auth risks
- mobile issues
- simplest improvement before launch

Give me the top 5 fixes only, ordered by risk.
```

## 7. Day 60 Recap Builder

```text
Use these daily logs: [paste logs].

Build a Day 60 recap outline with:
- biggest revenue jumps
- biggest follower jumps
- best weeks
- worst mistakes
- best proof moments
- offers launched
- tools built
- lessons worth teaching
- next 30-day plan
```
