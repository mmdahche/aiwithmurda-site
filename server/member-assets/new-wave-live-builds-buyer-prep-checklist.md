# New Wave Live Builds - Buyer Prep Checklist

## Bring This Before The Room

- One workflow you want to understand, copy, or improve.
- A short before-state note describing how the workflow happens now.
- Screenshots, files, links, or examples that show the current process.
- The tool stack involved.
- One clear reason the workflow matters to money, audience, time, or operations.

## Before-State Prompt

Use this before the room:

```text
I want to bring this workflow into a live AI build room.

Workflow:
Current manual steps:
Where it breaks or wastes time:
Who needs the output:
What a useful first version would do:
How I would prove the build worked:
What I do not want to overbuild:
```

## During The Room

Track these notes while watching:

- What did Murad refuse to build?
- What prompt moved the work forward?
- What test exposed the weak point?
- What proof made the change believable?
- What would you copy for your own workflow?

## After The Room

Within seven days:

1. Rewrite the build brief for your own workflow.
2. Run the first scoping prompt.
3. Create a before/after receipt, even if the build is rough.
4. Publish or send one proof-backed CTA.
5. Save the strongest question for the next live room.
