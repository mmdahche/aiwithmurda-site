# START HERE — Skill Authoring Kit

You bought the meta-skill: the discipline for turning your repeated
workflows into skills your AI agent reliably finds and fires. Stop
installing other people's skill packs and start minting your own — every
other pack you ever buy gets more valuable once you can author.

## Your first win in 3 steps (~20 minutes)

1. **Install the authoring skill:** copy `install/claude-code/write-a-skill.md`
   into your Claude Code skills/commands folder (or
   `install/codex/write-a-skill/SKILL.md` into `.agents/skills/`).
2. **Pick one workflow you've explained to your agent twice this week.**
   Twice = it recurs. That's your first skill.
3. **Say "write a skill for <that workflow>".** The authoring skill
   interviews you one question at a time, drafts the SKILL.md, runs the
   pre-ship audit, and tells you where to install it (see
   `payload/placement-guide.md` for the personal-vs-project decision).

Then read `payload/description-writing.md` — the single highest-leverage
five minutes in this kit. The description field is the only thing the skill
picker ever sees.

## Where everything is

- `payload/write-a-skill.md` — the authoring meta-skill (the product's core)
- `payload/skill-template.md` — the canonical blank template
- `payload/description-writing.md` — the load-bearing field, drilled
- `payload/placement-guide.md` — personal vs project scope, Claude Code + Codex paths
- `payload/validation-tdd.md` — RED/GREEN/REFACTOR behavioral validation for high-stakes skills
- `examples/` — two complete, installable worked examples (one simple, one pipeline-style)
- `install/` — drop-in layouts for Claude Code and Codex
- `VERIFY.md` — the checklist we ran before shipping; re-run it yourself

## The one rule

Don't codify what won't fire twice. A skill for a one-time task is clutter;
a skill for a weekly task is compound interest.
