# Design Studio Pack

**Ship UI that doesn't look like an LLM made it.**

Four clean-room design-discipline skills for founders and small teams building
web and product surfaces with AI coding agents. Turn the brand into a written
contract, audit against a deterministic anti-slop checklist, animate on
purpose, and critique screens the way a senior designer would.

| Skill | What it does |
|-------|--------------|
| **design-contract** | 9-section `DESIGN.md` brand contract (identity, palette, type, spacing, components, motion, voice, anti-patterns, verification) |
| **anti-slop-audit** | ~25-rule deterministic checklist that catches generic AI-generated UI before it ships |
| **motion-framework** | When to animate (enter/exit, duration tiers, easing, reduced-motion), when to sit still |
| **ui-critique** | Structured review protocol combining Nielsen heuristics with AI-slop tells |

## Time to first value

~15 minutes: install all four, draft a `DESIGN.md` for one product using
`design-contract`, then run `anti-slop-audit` on your current homepage screenshot.

## Install

**Quick path:** `bash install/setup.sh`

**Manual:** copy each file from `install/claude-code/` (Claude Code) or
`install/codex/<name>/SKILL.md` (Codex-style agents).

## Provenance

Clean-room authored 2026-07-16 by Murad Dahche. No third-party skill files
(atelier, impeccable, design-md, taste-design, or any other reference bundle)
were read, quoted, or paraphrased during authoring. All patterns are original
distillations of public design practice: Nielsen–Norman usability heuristics,
Material and Apple HIG motion guidance, WCAG 2.2 accessibility norms, and
common brand-system taxonomy (identity → tokens → components).
