# The First Week — what changes when your AI stops forgetting

A realistic day-by-day of installing Memory OS and living with it for a week,
so you know what "working" looks like before you start. No magic — just the
compound interest of written-down state.

## Day 1 — identity (20 minutes)

Fill in `templates/SOUL.md`: name, role, voice, five standing rules, red
lines. Merge `templates/AGENTS-ADDON.md` into your agent's workspace contract
and create the empty skeleton: `MEMORY.md` (from the template), `memory/`,
`handoffs/`. Install the three commands (`/handoff`, `/resume`, `/dispatch`).

First session after: the assistant introduces itself in the voice YOU wrote.
It knows nothing yet — knowing things is the week's job.

## Day 2 — the first handoff

Work a normal session. At the end: `/handoff`. Read the file it wrote —
fix anything mis-captured (early handoffs teach the assistant what you
consider load-bearing). Note the resume command at the bottom.

## Day 3 — the payoff moment

Open a fresh terminal. `/resume`. The assistant presents yesterday's state:
what was done, the open threads, the suggested next move — and waits. That's
the amnesia bridge working on day three.

During the day, say "remember this" about anything — watch it land in
`memory/2026-XX-XX.md` immediately, not at session end.

## Day 4 — the first correction becomes permanent

The assistant does something you don't like; you correct it. Same mistake
category twice = say "make that a rule." It writes a feedback memory with the
origin story and indexes it. That category of mistake is now patched — not
for this session, permanently.

## Day 5 — first dispatch

A well-scoped task you don't need to watch: `/dispatch` it with the briefing
template. Keep talking about something else. When the report returns, notice
the assistant reads the actual diff before relaying success — the report
contract at work.

## Day 7 — the first curation ritual (10 minutes)

Ask for the weekly memory curation. Watch it: read the week's daily notes,
propose 2-3 promotions to cluster files, phrase the index lines as triggers,
demote nothing yet (week one), correct anything wrong. Approve or adjust the
promotions — curation is a joint act the first few times.

## What you have after one week

- An assistant with your voice and your rules, stable across sessions
- Two or three cluster memories that already save re-explanation
- A handoff/resume rhythm that makes context death a non-event
- One permanent behavioral fix that would previously have been a weekly annoyance

## What to expect honestly

Week one memory is thin — the system's value compounds with the ritual, it
doesn't arrive installed. The failure mode to avoid is also the only real
one: skipping the weekly curation until the index is fiction. Ten minutes a
week is the entire maintenance bill.
