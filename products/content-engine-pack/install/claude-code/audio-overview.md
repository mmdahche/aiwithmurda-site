# Audio Overview

Transform **written source material** into a speakable script — not a read-aloud
of the PDF, but a guided listening experience.

## When to use

- Podcast episode outline from blog post or doc
- Voiceover for explainer video
- Audio summary of long report for busy stakeholders

## Inputs

- Source doc(s) or bullet notes
- Target length (minutes)
- Audience familiarity ( novice / practitioner )
- Tone (interview, solo narrator, co-host)

## Structure

1. **Cold open** (15–30s) — one sentence stakes; no "welcome to the show"
2. **Chapter beats** — 3–7 sections with timestamp targets
3. **Transition lines** — bridge between beats (short, spoken)
4. **Close** — one takeaway + optional CTA

## Script conventions

- `[PAUSE]` — half-second beat
- `[EMPHASIS: word]` — stress marker for VO
- Numbers spoken as words under 20; larger numbers rounded with "about"
- Acronyms spelled first use unless audience is expert

## Workflow

1. Extract 5–10 key claims from source
2. Order for listener logic (not document order)
3. Write conversational sentences ≤25 words
4. Read aloud — fix tongue-twisters
5. Time estimate: ~130 words/minute narration

## Ethics

- Don't add facts not in source
- Attribute quotes: "According to the report..."
- Mark omitted nuance in producer notes

## Output

```markdown
# Audio overview — [title]
Length target: Xm · Voice: solo
## Cold open
...
## Chapter 1 — [name] (~Xm)
...
## Producer notes
Omissions: ...
Sources: ...
```
